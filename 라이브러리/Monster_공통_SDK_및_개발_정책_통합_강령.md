## 🧭 제1장: 4대 핵심 공통 SDK (Software Development Kit)

3Monster 루트의 `라이브러리/` 폴더에 패키지 형태로 사전 구현되어 있으며, 개별 앱 개발 시 **복사 붙여넣기(Import)**하여 그대로 사용해야 하는 실체적인 파이썬 라이브러리 엔진입니다.

### 1. 🔒 인증 엔진 (`auth.py`) - 라이선스 및 HWID 기기 락킹
*   **핵심 기술**: Supabase REST API 직접 호출, Windows 기기 고유 하드웨어 식별자(HWID) 해싱 및 바인딩.
*   **원칙**:
    - **1PC 1Key 바인딩**: 최초 검증 시 사용자의 기기 HWID를 DB에 영구 귀속시켜 라이선스의 무단 배포 및 부정 사용을 원천 차단합니다.
    - **0.5초 타임아웃**: 서버 장애나 오프라인 상황에서 GUI 멈춤 현상(Deadlock)이 없도록 Supabase 커넥션에 강제로 0.5초 시간제한을 둡니다.

### 2. 🔄 업데이트 엔진 (`updater.py`) - 실시간 버전 대조 및 원격 배포
*   **핵심 기술**: HTTP 백그라운드 스트리밍 다운로드, 원격 패치 배치(Batch) 파일 프로세스 연동.
*   **원칙**:
    - 앱 부팅 시 Supabase의 최신 버전 테이블(`app_versions`)을 0.2초 이내 초고속 검증합니다.
    - 업데이트 감지 시 대화상자를 띄우고, 승인 시 백그라운드로 스트리밍 패치 파일을 내려받은 후 앱 프로세스를 스스로 종료하고 재부팅(자가 덮어쓰기)합니다.

### 3. 📝 로깅 엔진 (`logger.py`) - 5MB 롤링 로그
*   **핵심 기술**: Python `logging.handlers.RotatingFileHandler`.
*   **원칙**:
    - 프로그램 작동 로그 및 Traceback 에러 정보를 기록하되, 사용자 하드디스크의 무분별한 용량 과점유를 막기 위해 **단일 파일 크기 상한선 5MB** 및 백업본 최대 3개 초과 금지를 하드웨어 레벨에서 강제합니다.

### 4. 📁 내보내기 엔진 (`exporter.py`) - 파일 저장 및 탐색기 하이라이트
*   **핵심 기술**: OS 파일 매니저 연동 및 탐색기 선택(Focus) API.
*   **원칙**:
    - 브라우저 방식의 복잡한 버튼 다운로드 프로세스를 배제하고, 버튼 한 번으로 시스템 'Downloads' 폴더에 파일을 조용히 저장합니다.
    - **3단계 Prefix 파일명 전략**: ① 사용자 입력 검색 키워드(최우선) → ② 앱별 고정 기본 키워드 → ③ fallback 순으로 파일명을 자동 구성합니다.
    - 저장 직후 OS 파일 탐색기를 실행하여 방금 저장된 파일을 파란색으로 선택(Focus)해주어 즉시 사용 가능한 상태를 지원합니다.

## ⚖️ 제2장: 3대 핵심 개발 정책 (Development Policies)

개별 앱이 허브(웹 대시보드)와 상호 유기적으로 작동하고 일관된 브랜드 아이덴티티를 갖도록 규정한 개발 가이드라인입니다. 이는 SDK 코드로 제공되는 것이 아닌 **"구현 규정"**입니다.

### 1. 📂 사용자 설정 경로 독립화 정책 (Settings Isolation Policy)
*   **답변 및 정의**: 본 사안은 별도의 설정을 다루는 공통 SDK를 만들지 않고 **[정책 가이드]**로 통제합니다.
    *   *이유*: 앱마다 다루는 설정(키워드 수집 한도, 카페 URL 리스트, 댓글 템플릿 등)의 구조가 완전히 달라 획일적인 SDK로 구현 시 불필요한 무게만 차지하게 됩니다.
*   **정책**:
    - 모든 개별 앱의 사용자 설정 정보(`user_settings.json`)는 프로그램 설치 경로가 아닌 반드시 윈도우 시스템 공통 경로인 **`%APPDATA%/{AppName}/`** 경로에 위치시켜야 합니다.
    - 이렇게 분리해야만 `updater.py`에 의해 프로그램 설치 폴더 전체가 최신 파일로 덮어쓰기 및 업데이트될 때 사용자의 옵션 데이터가 영구 유실되는 불상사를 막을 수 있습니다.

### 2. 🖥️ 화면 해상도 및 배치 오프셋 정책 (Window Geometry Policy)
*   **정책**:
    - **인증 창**: `auth_gui.py`가 구현하는 최초 라이선스 인증 창은 무조건 모니터 화면의 **정중앙(Center)**에 노출되어 사용자의 시선을 집중시킵니다.
    - **메인 대시보드**: 개별 앱 메인 대시보드는 해상도 **`900x700`** 표준 크기를 적용하며, 기동 시 모니터 좌상단 모서리로부터 가로 40px, 세로 40px의 오프셋 간격(`+40+40`)을 두고 출력해야 합니다. 브라우저 창이나 다른 활성 윈도우와 완전히 겹쳐 숨겨지는 문제를 미연에 방지합니다.

### 3. 📋 에러 발생 시 무의존성 클립보드 복사 정책 (Clipboard Error Copy Policy)
*   **정책**:
    - 프로그램 오작동 및 치명적 에러 발생 시, 외부 파이썬 클립보드 패키지 의존성 없이 표준 Tkinter API를 백그라운드로 제어하여 **에러 전문과 하드웨어 HWID 사양이 결합된 디버그 리포트**를 사용자의 클립보드에 실시간 강제 복사합니다.
    - 팝업 창을 통해 사용자에게 "에러 내용이 복사되었습니다"를 고지한 후 3Monster 통합 허브의 1:1 고객센터 링크로 직접 유도하여 유저가 `Ctrl + V` 한 번만으로 정밀 에러 접수를 할 수 있도록 UX를 일원화합니다.
